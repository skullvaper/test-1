## 1. АНАЛІЗ ПОТОЧНОГО СТАНУ

### 1.1 Існуюча архітектура

**Основні компоненти:**
- **Frontend**: React 18 + TypeScript + Vite + Tailwind CSS
- **State Management**: Zustand (useGame.ts + expedition/store.ts)
- **Backend**: Supabase (Edge Functions + PostgreSQL + RLS)
- **Authentication**: Telegram Mini App SDK
- **Payments**: Telegram Stars + Supabase Edge Functions

**Існуючі системи в `src/expedition/`:**

## 4. ІНТЕГРАЦІЯ З ГОЛОВНОЮ ГРОЮ

### 4.1 Взаємодія систем

**Expedition → Main Game:**
```
Academy Level → Bonus to:
  - Passive XP multiplier
  - Artifact drop rate
  - Energy regeneration rate
  
Historical Prestige → Bonus to:
  - Main game prestige points
  - Unique artifact drops
  
Museum Collections → Bonus to:
  - Currency multiplier
  - Referral bonus
```

**Main Game → Expedition:**
```
Prestige Level ≥ 2 → Unlock Expedition
  
Expedition unlocks → Access to:
  - Academy Timeline
  - Historical expeditions
  - Museum system
  
Game currency → Can be spent in Expedition
Expedition currency → Can be converted to game resources (rare)
```

### 4.2 Resource Flow

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  Main Game      │────▶│  Expedition      │────▶│  Museum         │
│  (Energy, XP)   │     │  (Karbovanets)   │     │  (Prestige)     │
└─────────────────┘     └──────────────────┘     └─────────────────┘
        │                      │                        │
        ▼                      ▼                        ▼
   [+5% XP/tap          [+bonus to              [+Historical
    when energy]         expedition success]      Prestige bonus]
```

```sql
CREATE POLICY "Academy: users can CRUD own"
ON academy_progress FOR ALL
USING (
  telegram_id IN (
    SELECT (init_data->>'user'->>'id')::bigint 
    FROM auth.users 
    WHERE id = auth.uid()
```
